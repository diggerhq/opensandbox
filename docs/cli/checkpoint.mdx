## Creating a Checkpoint

{/* oc checkpoint create. What gets captured (memory, disk, processes). */}

```bash
oc checkpoint create sb-abc123 --name before-migration
```

## Forking from a Checkpoint

{/* oc checkpoint spawn. Creates a new sandbox from saved state. Parallel exploration pattern. */}

```bash
# Spawn two independent sandboxes from the same checkpoint
ID1=$(oc checkpoint spawn cp-7f3a1b2c --json | jq -r '.sandboxID')
ID2=$(oc checkpoint spawn cp-7f3a1b2c --json | jq -r '.sandboxID')

# Run different experiments
oc exec $ID1 --wait -- ./experiment-a.sh
oc exec $ID2 --wait -- ./experiment-b.sh
```

## Checkpoint vs Hibernate

{/* When to use which:
  - Checkpoint: branching, forking, testing multiple approaches
  - Hibernate: pause and resume the same sandbox, cost savings
*/}

| | Checkpoint | Hibernate |
| --- | --- | --- |
| Original sandbox | Keeps running | Stopped |
| Can fork / clone | Yes | No |
| Use case | Branching, testing | Pause and resume |
