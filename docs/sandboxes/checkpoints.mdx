## What is a Checkpoint

{/* Named snapshot of a running sandbox's full state (memory, disk, processes). Fork new sandboxes from it — each fork starts exactly where the checkpoint left off. Like git commits for VMs. */}

## Checkpoints vs Hibernation

{/* Key difference:
  - Hibernation pauses and resumes the SAME sandbox
  - Checkpoints create NEW sandboxes from saved state
  - A sandbox can have many checkpoints; hibernation is a single pause/resume
*/}
