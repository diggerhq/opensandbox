---
title: 'Create Reservations'
api: 'POST /api/capacity/reservations'
---

<Info>
  **Coming soon.** Reserved capacity is not yet available. See the

Reserve capacity across one or more 15-minute UTC intervals. Atomic across
the request — either every interval is booked or nothing is. See
[Reserving and cancelling](/reserved-capacity/reserving-and-cancelling) for
the full write contract.

Send an `Idempotency-Key` header so retries return the original result. Same
key with a different body returns `idempotency_key_conflict`.

<ParamField body="intervals" type="array" required>
  One entry per 15-minute interval to reserve.
</ParamField>

<ParamField body="intervals[].startsAt" type="string" required>
  Interval start. RFC 3339, UTC, aligned to 15 minutes (`:00`, `:15`, `:30`, `:45`).
</ParamField>

<ParamField body="intervals[].endsAt" type="string" required>
  Interval end. Must equal `startsAt + 15 minutes`. Multi-interval spans are rejected.
</ParamField>

<ParamField body="intervals[].capacityGb" type="integer" required>
  Positive integer. Number of 1 GB blocks to create for this interval.
</ParamField>
