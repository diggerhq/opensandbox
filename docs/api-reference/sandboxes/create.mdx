---
title: 'Create Sandbox'
api: 'POST /api/sandboxes'
---

Create a new sandbox.

<ParamField body="templateID" type="string">
  Template name (default: `"base"`)
</ParamField>

<ParamField body="timeout" type="integer">
  Idle timeout in seconds (default: `300`)
</ParamField>

<ParamField body="cpuCount" type="integer">
  CPU cores. Must match an allowed tier: `1`, `2`, `4`, `8`, or `16`. If omitted but `memoryMB` is set, inferred automatically.
</ParamField>

<ParamField body="memoryMB" type="integer">
  Memory in MB. Must match an allowed tier: `4096`, `8192`, `16384`, `32768`, or `65536`. If omitted but `cpuCount` is set, inferred automatically.
</ParamField>

The allowed CPU/memory combinations are:

| Memory | vCPU |
| --- | --- |
| 4096 MB (4 GB) | 1 |
| 8192 MB (8 GB) | 2 |
| 16384 MB (16 GB) | 4 |
| 32768 MB (32 GB) | 8 |
| 65536 MB (64 GB) | 16 |

If both `cpuCount` and `memoryMB` are provided, they must match one of these tiers.

<ParamField body="envs" type="object">
  Environment variables as key-value pairs
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary key-value pairs
</ParamField>

<ParamField body="image" type="object">
  Declarative image manifest (see [Image builder](/sandboxes/templates#image-configuration))
</ParamField>

<ParamField body="snapshot" type="string">
  Name of a pre-built snapshot for instant boot
</ParamField>

<ResponseExample>
```json 201
{
  "sandboxID": "sb-abc123",
  "status": "running",
  "region": "use2",
  "workerID": "w-use2-abc123"
}
```
</ResponseExample>
